1. B
   解析：抽象硬件架构用于建立统一的硬件视角，帮助理解 SIMD 编程模型、分析数据流，并为后续编写算子做准备；具体产品规格仍需要查阅目标架构的官方文档。

2. A
   解析：Scalar 主要负责地址计算、流程控制、分支控制和任务发射，不负责大批量逐元素计算或数据搬运。

3. C
   解析：Vector 适合对一批元素执行相同或相近的逐元素计算；保存数据属于存储单元职责，搬运数据属于 DMA 职责。

4. A
   解析：GM 通常保存 Device 侧全局可见的数据，LM 是 AI Core 片上本地存储，常用于保存当前分片数据和中间结果。

5. B
   解析：从计算数据流角度看，输入输出通常位于 GM，当前分片搬入 LM 后由 Vector 或 Cube 计算，结果再写回 GM。
