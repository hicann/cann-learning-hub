2. √
解析：HIXL 提供单边零拷贝通信机制，零拷贝实现用户内存间的直接数据传输，避免冗余数据搬运，不仅可以降低内存带宽占用，还可以减少内存容量消耗。

3. B
解析：HIXL Engine 作为核心传输引擎，提供基础传输接口，支持 D2D、D2H、H2D 等多种类型内存传输，同时兼容 HCCS、RDMA 等多种传输协议。LLM-DataDist 是基于 HIXL Engine 构建，提供携带 KV Cache 语义的数据传输接口。

4. C
解析：大模型 PD 分离场景中的 KV Cache 跨节点传输是 HIXL 的典型应用场景，HIXL 可实现 P/D 节点之间高效的 KV Cache 传输。选项 A 属于集合通信场景，应使用 HCCL；选项 B 和 D 不涉及跨节点数据传输。

5. A、B、C
解析：HIXL 的核心优势包括：支持单边零拷贝通信机制、屏蔽硬件差异兼容多链路、极简 API 设计深度适配开源生态。选项 D 错误，HIXL 不自动解析上层模型结构语义，业务层需要负责解释 KV Cache 或参数页的语义。
