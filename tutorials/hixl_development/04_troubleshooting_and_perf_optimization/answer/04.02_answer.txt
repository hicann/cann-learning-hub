课后练习答案

1. （判断题）当 HIXL 接口调用失败时，可以通过 `aclGetRecentErrMsg` 获取错误码、接口调用堆栈和相关日志参数打印。
答案：正确
解析：本节给出了示例代码，说明 HIXL 接口调用失败时，可在异常分支通过 `aclGetRecentErrMsg` 打印错误码、调用堆栈以及相关日志参数信息，这是定位问题的第一步。

2. （判断题）建链报错时，只要 client 侧已经打印出 TraceBack，就无需结合 server 侧日志继续分析。
答案：错误
解析：本节明确指出，HIXL 应用开发分为 client 侧和 server 侧，定位问题时应同时结合两端日志分析，不能只关注一侧的报错；尤其是建链类问题，往往需要对比 client 和 server 的关键信息。

3. （单选题）当仅从 HIXL 的错误堆栈仍无法定位问题根因时，下一步最适合进一步查看哪个目录下的底层日志？
答案：A
解析：当 `aclGetRecentErrMsg` 提供的 HIXL 错误堆栈不足以定位问题时，需要继续查看底层 HCCL 或 RUNTIME 日志。本节说明应用类日志默认保存在 `$HOME/ascend/log/plog/`。

4. （单选题）在“建链超时时间非法”案例中，以下哪条报错信息最直接表明根因是建链超时时间传参错误？
答案：A
解析：在该案例中，`aclGetRecentErrMsg` 直接打印出了 `timeout_in_millis:0 must > 0`。这条信息明确指出 `Connect` 接口传入的超时时间 `0` 非法，因此可以直接锁定根因为参数错误。

5. （多选题）在“建链两端链路不一致”案例中，以下哪些分析结论或处理方法是正确的？
答案：A、B、D
解析：对比两侧 `LINK_ERROR_INFO` 中的 IP 信息，可以发现链路是否一致，因此 A 正确；本节给出的解决方案是检查 `HCCL_INTRA_ROCE_ENABLE`，两端需要同时配置为 1 或同时都不配置，因此 B 正确；`MyRole`、`Status`、`TlsStatus` 等字段也都是判断链路状态的重要线索，因此 D 正确。仅凭 `dest_port` 一致并不能排除链路配置不一致的问题，所以 C 错误。
