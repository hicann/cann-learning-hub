1. 纯 Kernel 耗时 12.9 us，对于 50K 输入的 Gather 操作属于合理范围
2. Block Dim = 40，充分利用 910B 的 AI Core
3. MTE2 主导 (60.8%) 符合 Gather 类算子特征（非连续读取）
4. icache_miss_rate = 0%，指令缓存命中完美
5. 核间负载均衡良好

注意事项:
- torch.as_strided() 本身是视图操作（零拷贝），不触发 NPU Kernel
- 本测量针对 .contiguous() 物化场景，即实际需要数据搬移的情况
- 自定义 Ascend C Kernel (as_strided) 通过 aclnn API 直接调用时
  走独立路径，性能特征与 torch_npu 内置 Slice 可能不同
- 当前 profiler 采集到的是 torch_npu dispatch 到的 Slice kernel，
  非自定义 Ascend C as_strided kernel
